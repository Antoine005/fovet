Souverain par conception

Pourquoi Ardent ?

Dans les systèmes critiques, chaque dépendance est un risque. Ardent est conçu pour en éliminer le maximum.

🚫

Aucun cloud étranger

Le traitement s'effectue en local, directement sur le microcontrôleur. Vos données ne quittent jamais votre infrastructure — elles ne peuvent pas être interceptées, revendues ni soumises à des législations extraterritoriales.

CLOUD Act · FISA : inapplicables par conception
⚡

Temps réel garanti

Moins d'une milliseconde par échantillon à 80 MHz. Dans les systèmes embarqués critiques, une anomalie non détectée en temps réel peut compromettre une mission, une machine ou une vie. Ardent ne fait aucune concession sur la latence.

< 1 ms · déterministe · pas de jitter RTOS
📦

Empreinte minimale

Zéro allocation dynamique. Moins de 4 KB RAM par détecteur. Ardent fonctionne sur les microcontrôleurs les plus contraints — là où un RTOS lourd ou un modèle ML classique ne passerait pas. Optimisé pour les MCU d'entrée de gamme comme pour les cœurs haute performance.

< 4 KB RAM · stack only · 0 malloc
🔍

Auditabilité totale

Code C99 pur, lisible, sans dépendance opaque. Vos équipes sécurité peuvent auditer chaque ligne. Aucun blob binaire, aucune bibliothèque tierce mystérieuse, aucune surface d'attaque cachée. Licence duale : open-source pour les projets publics, commerciale pour les usages industriels.

C99 · open-source · licence duale

À propos

Pourquoi Ardent existe

Les systèmes embarqués critiques méritent des outils à la hauteur de leurs contraintes. Ardent est né du constat qu'aucune solution existante ne combinait à la fois la souveraineté, la légèreté et l'auditabilité nécessaires aux environnements industriels et de défense.

Conçu par un ingénieur issu du monde aéronautique et de l'embarqué, Ardent privilégie l'ingénierie de précision à la complexité superflue — chaque kilooctet compte, chaque milliseconde compte.

🇫🇷

Souveraineté numérique

Infrastructure française, code ouvert, aucune dépendance étrangère critique.

🎯

Ingénierie de précision

Chaque ligne de code est justifiée. Pas de sur-ingénierie, pas d'abstraction inutile.
